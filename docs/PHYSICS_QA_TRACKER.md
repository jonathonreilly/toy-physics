## QA Rules

- Add only issues that are specific and checkable.
- Link the exact file, note, script, log, or commit involved.
- Prefer diagnosed causes over vague suspicion.
- When an issue is resolved, move it to `Recently Resolved` with the retest used.
- Do not treat a retrospective literature resemblance as validation.
