wastefully cycled against by narrow-edit campaigns.

## Why citation cycles matter

Per the audit lane policy
([`docs/audit/README.md`](audit/README.md)), a citation cycle in the
graph forces every node in the cycle to `retained_pending_chain`
regardless of audit verdict. Each cycle needs one auditor-designated
'see also' edge stripped, or one node promoted with an explicit
`cycle_break_required` re-audit. Until that happens, none of the
in-cycle rows can become `retained` by chain closure alone — every
cycle member is permanently blocked from retained-grade.

The audit queue surfaces cycle-break targets at
[`docs/audit/AUDIT_QUEUE.md`](audit/AUDIT_QUEUE.md) under the
"Citation cycle break targets" section, with the full per-cycle
instruction text and break-target ranking in
[`docs/audit/data/audit_queue.json`](audit/data/audit_queue.json) under
the `cycle_break_targets` field, and the cycle inventory itself in
[`docs/audit/data/cycle_inventory.json`](audit/data/cycle_inventory.json).

## Repair-class distribution (top 25)

| audit_status | count |
|---|---:|
| `unaudited` | 25 |
| `audited_conditional` | 0 |

| repair_class on the cycle entry | count |
|---|---:|
| `cycle_break_required` | 25 |

**All 25 highest-leverage cycle-break primary targets are currently
`unaudited` and `critical`.** None has yet been audited; therefore none
carries an auditor-assigned per-row repair class
(`missing_bridge_theorem` / `scope_too_broad` /
`missing_dependency_edge` / `dependency_not_retained` /
`runner_artifact_issue` / `compute_required` / `other`) in its
`notes_for_re_audit_if_any`. The repair class on each cycle entry is
the queue-assigned `cycle_break_required`, which is the audit lane's
fixed signal that the cycle requires a break re-audit per the policy
in [`docs/audit/README.md`](audit/README.md).

The fixed per-cycle instruction template, present on every entry in
`cycle_break_targets`, asks the auditor to:

1. Re-audit the primary break target with explicit cycle-break
   instructions naming the co-cycle citations as informational/'see
   also' references rather than load-bearing dependencies.
2. If the chain closes without those citations, return `audited_clean`
   and name the non-load-bearing co-cycle links in the rationale; a
   separate source-graph repair pass then strips or rewrites those
   markdown links before `effective_status` can leave
   `retained_pending_chain`.
3. Otherwise return `audited_conditional` with
   `repair_class=missing_dependency_edge` naming the node that should
   be promoted upstream.

So a cycle-break re-audit is structurally either a "see also" markup
fix (cheapest possible repair) or a `missing_dependency_edge`
verdict (also a bounded source-note edit). Neither path is
`missing_bridge_theorem`-class work. The cycle-break track is
therefore a low-cost / high-leverage track relative to the
bridge-theorem-derivation track that gates the plaquette / alpha_s
chain (see PR #988).

## High-leverage low-cost candidates

Because all 25 top primary break targets are `unaudited` and the
fixed cycle-break instruction template channels the verdict into either
`audited_clean` (with a 'see also' source-graph repair note) or
`audited_conditional` with `repair_class=missing_dependency_edge`
(also a bounded source-note edit), **every one of the 25 top primary
break targets is a low-cost / high-leverage candidate by audit-lane
policy**.

None of the top 25 carries `missing_bridge_theorem` on its current
ledger row. (None could — they are all `unaudited`.) The
`missing_bridge_theorem` repair class only appears on rows that have
already been audited and returned `audited_conditional` with that
specific repair class. It does not appear on any of the 305
cycle-entry primary break targets at write time except for the 4
`audited_conditional` entries below the top 25, all of which are
`dependency_not_retained` rather than `missing_bridge_theorem`.

So at write time:

- **0 of 25** top cycle-break primary targets are blocked by
  `missing_bridge_theorem` repair work.
- **25 of 25** top cycle-break primary targets are tractable by a
  single bounded cycle-break re-audit per the fixed instruction
  template.

This is the inverse cost-shape from the plaquette / alpha_s chain
frontier surveyed in PR #988, where 4 of 5 conditional frontier rows
were `missing_bridge_theorem` and only 1 was a bounded narrow-edit
(scope split) repair. Cycle-break work as a class is far cheaper per
unblocked transitive descendant than chain-frontier bridge-theorem
work.

## Recommendation for future campaigns

- **Attack the top-25 cycle-break primary targets first.** Each of
  the 17 unique top-25 break targets is `unaudited`, `critical`, and
  channeled into a bounded repair (either `audited_clean` with a
  source-graph 'see also' edit or `audited_conditional` with
  `repair_class=missing_dependency_edge`). None requires
  bridge-theorem derivation work to be audited under the cycle-break
  template.
- **Highest single-row leverage:** spawn cycle-break re-audit cycles
  against
  [`area_law_coefficient_gap_note`](AREA_LAW_COEFFICIENT_GAP_NOTE.md)
  and [`alpha_s_derived_note`](ALPHA_S_DERIVED_NOTE.md) first. Each
  resolves 3 of the top 25 cycles in a single re-audit and each shares
  primary-break-target status with cycles in the
  `max_desc >= 642` band.
- **Do not** spawn cycle-break re-audits against the 4 cycles whose
  primary break target is already `audited_conditional` with
  `dependency_not_retained` (cycles 0060, 0091, 0104, 0119, all
  pointing at `neutrino_mass_reduction_to_dirac_note` per the live
  queue at write time). Their repair surface is already specified by
  the existing audit verdict and does not benefit from a fresh
  cycle-break re-audit instruction; the cheapest action there is to
  audit the named upstream dependency the existing repair note calls
  out, not to re-audit the conditional row again.
- **The remaining 280 unaudited critical primary break targets** are
  each individually tractable but lower per-row leverage than the top
  25. Recommend: clear top 25 first, then fan out to 26-100 by
  `max_desc` order before touching the long tail.
- **Do not promote** any cycle-break primary target without the
  cycle-break-aware re-audit instruction. A standard fresh-context
  audit on these rows will not return `audited_clean` because the
  citation cycle prevents chain closure regardless of the per-row
  derivation quality.
