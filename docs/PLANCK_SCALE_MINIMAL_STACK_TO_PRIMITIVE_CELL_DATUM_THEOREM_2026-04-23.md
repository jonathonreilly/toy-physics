information principle, but that is a later step, not proved here.

## Proof sketch

### 1. The front-door stack already fixes a local algebra on a physical `Z^3`

The front-door memo fixes `Cl(3)` as the local algebra and `Z^3` as the
substrate, with finite local lattice dynamics. The physical-lattice necessity
theorem then removes the regulator-family reinterpretation on the accepted
package surface. So the local substrate is not a disposable coordinate chart;
it is part of the physical ontology of the package.

This matters because datum language requires a real object to which the datum
belongs. After physical-lattice necessity, the primitive lattice cell is such
an object.

### 2. Derived time upgrades the local object from purely spatial to `3+1`

The anomaly/time theorem yields exactly one time direction. The scalar `3+1`
theorem records the local minimal `3 spatial + 1 time` block used by the live
package. So the local object relevant to the direct Planck lane is not merely a
three-bit spatial cube; it is a time-locked `3+1` cell.

### 3. The exact spatial carrier is already factorized as a three-bit cube

The site-phase / cube-shift intertwiner and `S_3` taste-cube decomposition fix
the exact spatial local cube carrier as

`C^8 = (C^2)^{⊗ 3} = C^2_x ⊗ C^2_y ⊗ C^2_z`.

So the spatial part of the local object is already a native three-factor
two-state carrier.

### 4. Appending the unique time factor gives the exact primitive local cell

The unique derived time direction contributes one temporal two-state factor
`C^2_t` on the minimal local `3+1` block. Therefore the exact time-locked local
cell is

`H_cell = C^2_t ⊗ C^2_x ⊗ C^2_y ⊗ C^2_z ~= C^16`.

This is the same factorized cell used by the direct Planck route, but now
earned from the front-door stack plus already-earned package consequences.

### 5. Why this is the native object for source-free local state assignment

Once the lattice is physical and the exact local cell object is fixed, the
phrase "source-free local state" must refer to a state on that primitive cell
object if it refers to anything local at all. Otherwise the state would be
attached to:

- a regulator family not in the accepted stack, or
- a presentation-dependent carrier not identical to the primitive cell.

Both options reintroduce exactly the ontology the physical-lattice and
same-object package moves were supposed to exclude.

Therefore the direct lane is entitled to phrase its remaining open question as:

`what is the source-free datum on H_cell?`

That is the primitive-cell datum formulation.
