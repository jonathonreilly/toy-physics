## Question

What from the rejected PF review packet is worth preserving on `main`?

## Short answer

Keep the **route decomposition** and **dependency analysis**.

Reject the theorem-grade claim that the current exact bank already certifies a
branch-wide negative Perron-Frobenius closure.

The branch contained useful structure about where the remaining constructive
work actually lives. It did **not** clear the repo's evidentiary bar for a
global or branch-wide closure theorem.

## What is preserved

Three ideas from the review branch are worth keeping as route history.

### 1. The PF lane has a real work order

The surviving useful contribution from the branch is that the PF search no
longer has to be treated as one undifferentiated slogan about a global
selector.

The honest work order is:

1. strengthen the Wilson-side parent / intertwiner surface;
2. try to land a real Wilson-to-sector descendant law, with the compressed
   Hermitian codomain as the clean first target;
3. continue the plaquette operator evaluator path at the explicit class-sector
   matrix-element level;
4. treat any common-state / PF compatibility theorem as downstream of those
   constructive steps, not as a substitute for them.

That ordering is worth preserving because it narrows future work onto specific
upstream seams.

### 2. The PMNS routes already converge back onto a Wilson-side primitive

The branch's route-audit work usefully sharpened the PMNS picture.

The strong and compressed PMNS-side paths do not define two independent future
programs. They both point back toward a missing Wilson-side charged
embedding/source-family construction, with the compressed Hermitian codomain
being the cleaner first target.

That is worth keeping as route history because it says future positive work
should try to build the descendant law first, not keep proliferating separate
PMNS subprograms.

### 3. Wilson remains the main positive reopening lever, but not the only blocker

The branch's dependency audit also captured one useful asymmetry:

- Wilson remains the most plausible future **positive** reopening lever;
- but the PF lane's remaining obstacles are not exhausted by Wilson alone.

Even a stronger Wilson-side story would still need a real descendant law and
real plaquette operator closure, rather than automatically yielding a global PF
selector.

That asymmetry is worth preserving because it keeps future salvage work from
overfitting to Wilson as a universal fix.

## What is explicitly rejected from the source branch

The following claim layer was **not** accepted onto `main`:

- branch-wide negative current-bank PF closure
- theorem-grade Wilson complete-closure certification
- theorem-grade PMNS scalar-discriminant frontier certification
- theorem-grade plaquette first-Hankel frontier certification
- the umbrella statement that no live positive PF route remains on the present
  bank

These claims were rejected because the branch's load-bearing runners did not
verify the actual branch objects at theorem grade.

## Why the closure packet was rejected

The review branch failed on evidence quality, not on ambition.

The central closure runner was a note-audit surface: it read prior notes,
looked for expected closure phrases, and then promoted a branch-wide verdict.

The representative Wilson / PMNS / plaquette certificate runners were better
than raw phrase checks, but still only proved generic small toy lemmas and then
checked whether earlier notes stated the desired frontier reduction.

So the branch did **not** establish that the current exact bank itself forces
those closures. It only established that the branch had a coherent internal
story about them.

## Re-entry conditions

The PF program should be reconsidered for live authority only if at least the
following are built:

1. object-level verifiers over the actual Wilson / PMNS / plaquette branch
   objects, rather than note-audit or toy-certificate surrogates;
2. a real Wilson-to-sector descendant theorem or an equally strong explicit
   plaquette operator evaluation law;
3. only after that, any branch-wide PF compatibility or selector claim.

Until those conditions are met, PF remains preserved here as route history and
attack-program material, not as accepted closure science.
