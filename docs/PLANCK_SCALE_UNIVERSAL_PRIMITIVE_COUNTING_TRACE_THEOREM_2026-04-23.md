## Question

Why should the elementary Planck coefficient be evaluated by normalized
primitive counting trace rather than by a dynamical state expectation?

## Bottom line

If the elementary coefficient is required to be:

1. attached to the primitive physical cell itself;
2. universal across admissible later vacua, Hamiltonians, embeddings, and
   prepared states;
3. local on the finite primitive event algebra;
4. positive and normalized;
5. additive on orthogonal primitive-event alternatives;
6. natural under relabeling of indistinguishable primitive event atoms;

then the coefficient functional is uniquely the normalized counting trace:

`tau(P) = rank(P) / dim(H_cell)`.

For the time-locked Planck cell,

`dim(H_cell) = 16`.

For the one-step worldtube packet,

`rank(P_A) = 4`.

Therefore

`tau(P_A) = 4/16 = 1/4`.

This is a uniqueness theorem for the primitive universal coefficient object
class. It does not say every physical density matrix is tracial. It says a
universal primitive-cell coefficient is not a dynamical state expectation unless
a separate theorem proves that the dynamical state always reproduces the same
counting trace.

## Setup

Let the primitive event algebra be generated by the sixteen atomic projectors

`E = {P_eta : eta in {0,1}^4}`.

Let `C(P)` be the elementary coefficient assigned to an event-frame projector
`P`.

The intended object is:

`(H_cell, E, P)`.

A dynamical expectation value instead has object class:

`(H_cell, E, P, iota, Omega, H_dyn, boundary data, ...)`.

That extra datum class is not primitive-cell data.

## Axioms for a universal primitive coefficient

For a universal primitive coefficient, require:

### U1. Object-class locality

`C(P)` depends only on the primitive event frame and the projector `P`.

It does not depend on a global embedding, selected vacuum, Hamiltonian,
preparation, or boundary condition.

### U2. Positivity

If `P >= 0` is a projector, then

`C(P) >= 0`.

### U3. Normalization

The full primitive event frame has unit count:

`C(I) = 1`.

### U4. Finite additivity

If `P Q = 0`, then

`C(P + Q) = C(P) + C(Q)`.

This is the minimal count semantics for disjoint alternatives.

### U5. Atomic naturality

Primitive atoms of the bare source-free event frame are not distinguished by
extra labels. Therefore relabeling primitive atoms preserves the coefficient:

`C(g P g^-1) = C(P)`

for every event-frame relabeling `g`.

This is not the old `U(2)^4` basis-gauge claim. It is only naturality of the
finite primitive event atoms for the counting object.

## Theorem: normalized trace is unique

Under U1-U5,

`C(P) = rank(P) / 16`

for every event-frame projector `P`.

### Proof

Write an event-frame projector as a sum of atoms:

`P = sum_(eta in S) P_eta`.

By finite additivity,

`C(P) = sum_(eta in S) C(P_eta)`.

By atomic naturality, all primitive atoms have the same coefficient:

`C(P_eta) = c`

for every `eta`.

By normalization,

`1 = C(I) = sum_eta C(P_eta) = 16 c`.

So

`c = 1/16`.

Hence

`C(P) = |S| / 16 = rank(P) / 16`.

This is exactly the normalized counting trace.

## Corollary: the Planck packet coefficient

The one-step worldtube packet is

`P_A = 1_{N_evt = 1}`.

It contains the four Hamming-weight-one primitive events, so

`rank(P_A) = 4`.

Therefore

`C(P_A) = rank(P_A) / 16 = 4/16 = 1/4`.

## Why a dynamical state expectation is the wrong primitive object class

A dynamical state expectation has the form

`Tr(rho_dyn P_A)`.

Unless a theorem fixes `rho_dyn`, this depends on the extra datum selecting
`rho_dyn`.

The branch already has explicit same-cell witnesses:

- `Tr((I_16/16) P_A) = 1/4`;
- `Tr(rho_lt P_A) = 1/8` for a packet-light admissible local state.

So `Tr(rho P_A)` is not determined by the primitive cell and packet alone.

Therefore a generic dynamical state expectation cannot be the primitive
universal coefficient. At most it can be a representation after a separate
state theorem proves `rho_dyn` reduces to the normalized counting trace for this
purpose.
