# Poisson-Backed End-to-End Teleportation Audit

**Date:** 2026-04-25
**Status:** planning / integration artifact; not a manuscript claim surface
**Runner:** `scripts/frontier_teleportation_end_to_end_poisson.py`

## Scope

This note records the first end-to-end integration audit for the native
taste-qubit teleportation lane. It combines the earlier bounded pieces:

1. Poisson/CHSH ground-state construction;
2. deterministic trace to an encoded two-qubit logical taste resource;
3. Bell-basis teleportation channel;
4. Bob pre-record no-signaling audit;
5. directed causal two-bit Bell-record delivery.

The result remains ordinary quantum state teleportation only. It is not matter
teleportation, mass transfer, charge transfer, energy transfer, object
transport, or faster-than-light signaling.

It is also not a native preparation/readout derivation. The Poisson resource is
still obtained by offline ground-state extraction and tracing over cells and
spectator tastes.

## Script

Command:

```bash
python3 scripts/frontier_teleportation_end_to_end_poisson.py
```

The runner uses the default cases from
`scripts/frontier_teleportation_resource_from_poisson.py`:

- `1d_null`: `dim=1`, `side=8`, `mass=0`, `G=0`;
- `1d_poisson_chsh`: `dim=1`, `side=8`, `mass=0`, `G=1000`;
- `2d_poisson_chsh`: `dim=2`, `side=4`, `mass=0`, `G=1000`.

For each case it extracts the traced logical resource, then applies the
standard Bell-measurement and Pauli-correction teleportation channel to 134
input states: six Pauli-axis probes plus 128 random states.

## No-Signaling And Record Checks

For the passing Poisson cases, Bob's no-record state is independent of Alice's
input to numerical precision:

```text
1d_poisson_chsh max pairwise no-record distance: 2.220e-16
2d_poisson_chsh max pairwise no-record distance: 2.220e-16
```

The no-record Bob marginal may be biased relative to `I/2` for imperfect
resources:

```text
1d_poisson_chsh Bob marginal bias from I/2: 3.189e-02
2d_poisson_chsh Bob marginal bias from I/2: 1.213e-01
```

That bias is not input information and does not constitute signaling. The
input-dependent corrected state appears only after the Bell record is
delivered.

The directed record channel checks:

- no early delivery;
- delivery exactly once;
- explicit two-bit Bell label;
- branch correction after delivery.

Observed delivered-branch fidelities:

```text
1d_poisson_chsh: 0.998546
2d_poisson_chsh: 0.989495
```

## Acceptance Gates

The runner reports `PASS` for:

- null controls do not pass high-fidelity protocol;
- at least one Poisson case passes end-to-end gates;
- all passing cases keep Bob pre-record input-independence;
- all passing cases exercise four Bell outcomes;
- all passing cases have causal record delivery.

## Limitations

- The Poisson resource is still generated by offline diagonalization and
  deterministic tracing. This is not an operational preparation/readout
  protocol.
- The encoded Bell measurement and Bob correction are ideal logical
  operations.
- The causal channel transports an explicit classical record; it does not
  derive the record bits from the directed lattice or from an apparatus model.
- The result is small-surface only: default cases are `1D N=8` and `2D 4x4`.
- Passing the integrated audit does not promote the lane beyond planning /
  first artifact.
- No matter, charge, mass, energy, or macroscopic object is teleported.

## Status

The lane now has a single end-to-end protocol audit using Poisson-derived
encoded resources. The audit passes on the default positive Poisson cases and
fails the `G=0` null control as expected. Preparation/readout remains the main
open physical bridge.
