# Physics-First Attack Plan

**Date:** 2026-04-05  
**Status:** priority map for the next high-upside physics batches

## Goal

Push the strongest possible physics position before spending more effort on
write-up polish.

## Moonshot Rule

If we deliberately pivot toward a potential breakthrough branch, the branch
must satisfy a stricter rule than the normal weak-field program:

1. it must target one genuinely new observable the current program cannot
   already produce in a bounded way
2. it must keep a clear reduction path back to the retained weak-field lane
3. it must fail fast if the reduction breaks or if the new observable only
   appears through an obviously ad hoc artifact

Multiple moonshot branches are now allowed in parallel as a **portfolio**.

That is only useful if each branch still stands on its own:

- one branch cannot borrow evidential weight from another
- each branch needs its own script / log / note chain
- later unification only counts if the individual lanes are already retained

For the current program, the best such moonshot is no longer "black holes,
etc." in general. The branch is now better split into:

- an **exact-lattice control** that tells us which self-generated field ideas
  can survive the weak-field gates at all
- a **generated-family transfer** question asking whether that control can
  escape the exact lattice
- a **true propagating-field** question asking whether anything beyond
  same-site memory produces a qualitatively new observable without losing the
  weak-field lane

The earlier strong-field target has already landed in bounded form:

- absorbing horizon threshold is retained
- bidirectional no-return threshold is retained
- a first exact-lattice propagating-field pocket is retained, but only as a
  controlled phase-lag / overlap observable

The best next moonshot target is therefore narrower and more structural:

- first, push the exact-lattice wavefield lane and ask whether it can transfer
  off the exact lattice without losing the weak-field gates
- second, test a true propagating-field architecture on a retained grown
  family, not just the exact-lattice pocket
- third, push a sharper persistent-object / inertial-response architecture,
  now specifically targeting detector/readout localization on top of the new
  compact repeated-update source-object positive
- only fourth, return to stronger horizon / delay rhetoric if those
  architecture upgrades survive reduction

Operationally, this is now a portfolio:

1. exact-lattice wavefield / causal-field closure
2. retained-grown-family propagating-field closure
3. persistent object / inertial-response architecture
4. geometry-transfer discriminator and geometry-rule recovery
5. continuum / asymptotic hardening as the credibility lane
6. tiny falsification-first complex-action / absorptive analogue probes on retained families
7. strong-field trapping / horizon only after the first four stop advancing

The current repo already has strong bounded evidence in several lanes.
What is still missing on the physics side is not another narrative pass.
It is now one or more of these:

1. generated geometry that actually works
2. a real universality boundary on graph structure
3. a forced novelty or a real no-go theorem on the action / mass law
4. a clean 3D continuum / asymptotic closure
5. one moonshot architecture branch with an explicit fail-fast reduction check

Those are the physics bottlenecks that matter most now if the goal is not just
"a better version of the same story" but "a result that could genuinely
surprise a skeptical physicist."

The newest discriminator result also narrows one tempting lane:

- on the retained `kNN`-floor generated-family bridge, support rescue still
  beats the wavefield bridge on both sign and detector support
- the current generated-family bridge is therefore best read as
  **geometry-limited**
- that means repeated tuning of the same bridge family is no longer the main
  moonshot; the next serious generated-family step has to change the geometry
  rule enough to widen support, not just tweak the field update again
- the new downstream-reach fan now adds an even stronger no-go:
  it broadens static support a little, but it still does not make the
  wavefield lane matter, so the transfer bottleneck is not fixed by another
  reach-based geometry rule either

With Gate B actively being pushed elsewhere, the cleanest **parallel**
physics lane is now:

5. a real one-parameter-mass / additivity closure beyond the current
   test-particle family

That is the next place where the theory can either harden or fail honestly.

Current status on that parallel lane:

- a second-family additivity cross-check now exists on the retained 2D ordered lattice
- the remaining blocker is no longer family-local additivity
- it is persistent-pattern / inertial-response closure

## Operating Rule

Only count a batch as success if it does one of:

- forces a new retained prediction
- forces a real negative / no-go result
- closes a mechanistic gap that blocks the whole program

Do not count:

- another cleaner replay of known-law reproduction
- a memo-only claim without a real harness
- a branch that survives only in commit messages

For the moonshot branch, also do not count:

- visually dramatic path plots without a retained observable
- a trapping or horizon claim that does not come with a weak-field reduction
- an absorptive or bidirectional extension that immediately destroys the
  current Born / weak-field baseline and is then treated as "physics"

## Batch 1: Generated Geometry Closure

**Question**

Can we grow the geometry, rather than impose it, and still keep the good
physics?

**Why this is highest priority**

- It is the biggest remaining gap in the model.
- It is the clearest path from a toy model to a physics mechanism.
- It directly addresses the strongest “imposed geometry” criticism.

**Success criteria**

- one local growth rule
- one generated 3D family
- same-family Born, `k=0`, interference, decoherence, gravity sign, and
  `F~M` support
- no need to force a universal theorem

**What counts as progress**

- structured connectivity beats generic recomputed neighbors
- the generated family keeps the good observables without becoming a new
  unrelated architecture

**What does not count**

- another mixed growth replay with no clear generated-geometry signal
- a doc-only “pass” without a script/log/note chain

## Batch 2: Graph Universality Boundary

**Question**

How much graph structure is actually required before the retained gravity lane
breaks, and which observables fail first?

**Why this matters**

- The new inverse-problem harness made the graph story sharper, not looser.
- This is now one of the best places where the framework could produce a real
  universality / phase-boundary result instead of another branch memo.
- It is also where the current overclaims are easiest to kill honestly.

**Success criteria**

- one reconciled graph phase diagram across short damage ladders and retained-
  family sweeps
- one hardened random / structureless graph harness or a clean rejection
- one note separating:
  - sign survival
  - exponent fidelity
  - Born survival
  - graph-family dependence

**What counts as progress**

- a sharp threshold or a clean null-threshold result that survives multiple
  seeds
- a graph-family comparison that says what is generic and what is not

**What does not count**

- “almost any graph works” language without a harness
- one-seed positive anecdotes

## Batch 3: Forced Novelty Or No-Go On The Action Law

**Question**

Can the framework actually force the weak-field-linear phase valley, or can we
prove that it cannot without additional assumptions?

**Why this matters**

- This is where the current “Newton derived” temptation lives or dies.
- A real no-go result would be just as valuable as a stronger derivation.
- This is the cleanest route from “nice toy model” toward “forced structure.”

**Success criteria**

- one bounded derivation or no-go note
- one harness-backed statement wherever possible
- no circular mass-law closure

**What counts as progress**

- a clear statement of what the current framework can and cannot force
- a selector that survives adversarial reading
- or a rigorous barrier that rules out the stronger claim

**What does not count**

- a doc-only theorem claim
- more “uniqueness” rhetoric without closure of the weak link

## Batch 4: 3D Continuum / Asymptotic Closure

**Question**

What actually survives the refinement limit on the strongest ordered 3D
lattice lane?

**Why this matters**

- This is the best route to turning a strong finite-lattice result into a
  defensible continuum statement.
- It is the main way to answer the “small-grid artifact” critique.

**Success criteria**

- a clean `h` ladder with explicit same-family comparisons
- distance-law behavior stated with finite-size corrections, not hype
- a clear boundary between sign persistence, tail shape, and magnitude control

**What counts as progress**

- near-Newtonian finite-lattice behavior becomes better resolved
- the asymptotic statement gets narrower but more defensible
- the note separates what is stable from what is only slice-dependent

**What does not count**

- exact Newtonian wording unless the frozen data really supports it
- broad “all dimensions solved” language

## Batch 5: Moonshot Architecture Portfolio

**Question**

Can one or more minimal extensions beyond the current exact-lattice Green
control produce either:

- a geometry-generic recovery of the weak-field lane
- or a genuinely new causal-field observable

while still reducing back to the retained weak-field baseline?

**Why this matters**

- It is one of the few remaining directions with genuine step-change upside.
- The first partial generated-family support recovery means the architecture is
  no longer all-or-nothing; there is now a concrete recovery path to test.
- If it fails cleanly, that also tells us where the current framework stops.

**Minimal target**

Do **not** target "black holes" broadly.

Each individual probe should still target exactly one of:

- generated-family recovery of `TOWARD` + `F~M` + support breadth
- true propagating-field lag / phase observable beyond same-site memory
- escape fraction / no-return only if the above two are stalled

And use exactly one minimal extension:

- connectivity-side support recovery on the retained generated family
- or a true propagating-field update on the exact lattice
- or, only as a secondary branch, absorptive / bidirectional / retarded-field
  strong-field probes with an explicit reduction check

**Success criteria**

- one script / log / note chain
- one new observable with a qualitative threshold, clean null, or clear
  weak-field recovery/failure boundary
- one explicit weak-field reduction check

**Fail-fast criteria**

Stop the branch quickly if:

- the extension cannot recover the retained weak-field sign / `F~M` lane in
  its weak-coupling or low-compactness limit
- the effect only appears by violating the current Born / normalization
  baseline in a trivial way
- the result is branch-fragile and cannot be frozen as one bounded observable

**What counts as progress**

- a generated-family rescue that restores sign, support, and `F~M` together
- a retained trapping or no-return threshold
- a retained retarded-field or delay-proxy observable with a clean
  instantaneous-limit reduction
- a retained no-go note saying the minimal extension cannot preserve the
  weak-field lane
- a sharp statement that amplitude loss, bidirectionality, or backreaction is
  the true bottleneck

**What does not count**

- commit-message-only moonshot narrative
- three extension anecdotes with no frozen artifact chain
- "qualitatively black-hole-like" language without a retained threshold
- broad gravitational-wave rhetoric without a retained bounded observable and
  an explicit scheduling-artifact caveat

1. Generated geometry closure

4. 3D continuum / asymptotic closure
5. Moonshot strong-field branch

## Recommended Parallel Non-Gate-B Order

1. Graph universality boundary
2. One-parameter mass / additivity closure
3. 3D continuum / asymptotic closure
4. 4D asymptotic support only after the first three are tighter

This is the best order when generated geometry is already being worked in
parallel elsewhere.

That first parallel item now means:

- harden the structureless-DAG lane and the graph phase-diagram reconciliation
  first
- only then return to persistent-pattern / inertial-response closure
- only then do a third-family additivity check if a cheap reusable family exists

That order improves the physics position fastest:

- generated geometry closes the biggest mechanistic gap
- graph-boundary work can produce either a real universality phase diagram or a
  decisive bounded negative
- a no-go theorem prevents wasted effort on circular derivation claims
- asymptotic closure reduces the artifact / finite-size critique

## What to Deprioritize

- more kernel fishing without a new physics question
- more action sweeps without a closure target
- more 4D work before the 3D asymptotics are tighter
- more doc polishing that is not attached to a new retained artifact
- more localization hunting on retained `3D h=0.5` or retained `2D`
- BMV promotion before a real artifact chain exists

## Operating Rule

If a task does not clearly improve one of the three closures above, defer it.
