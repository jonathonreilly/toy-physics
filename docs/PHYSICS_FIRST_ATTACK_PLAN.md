# Physics-First Attack Plan

**Date:** 2026-04-04  
**Status:** priority map for the next three physics batches

## Goal

Push the strongest possible physics position before spending more effort on
write-up polish.

The current repo already has strong bounded evidence in several lanes.
What is still missing on the physics side is not another narrative pass. It is
one or more of these three closures:

1. generated geometry that actually works
2. one coherent mechanism across architectures
3. a clean 3D continuum / asymptotic closure

Those are the physics bottlenecks that matter most now.

## Batch 1: Generated Geometry Closure

**Question**

Can we grow the geometry, rather than impose it, and still keep the good
physics?

**Why this is highest priority**

- It is the biggest remaining gap in the model.
- It is the clearest path from a toy model to a physics mechanism.
- It directly addresses the strongest “imposed geometry” criticism.

**Success criteria**

- one local growth rule
- one generated 3D family
- same-family Born, `k=0`, interference, decoherence, gravity sign, and
  `F~M` support
- no need to force a universal theorem

**What counts as progress**

- structured connectivity beats generic recomputed neighbors
- the generated family keeps the good observables without becoming a new
  unrelated architecture

**What does not count**

- another mixed growth replay with no clear generated-geometry signal
- a doc-only “pass” without a script/log/note chain

## Batch 2: Architecture / Action Unification

**Question**

Are the spent-delay and valley-linear lanes two different theories, or two
effective descriptions of one mechanism at different geometric scales?

**Why this matters**

- This is the main reason the current story still looks split.
- Reviewers will keep asking which action is “the” model.
- A clean bridge here would materially improve both the physics and the
  credibility.

**Success criteria**

- one explicit comparison family that varies regularity
- one clear statement of which action wins where
- one honest bridge hypothesis that is either strengthened or rejected

**What counts as progress**

- regular geometry favors valley-linear
- more irregular geometry favors spent-delay
- the crossover looks scale-dependent rather than arbitrary

**What does not count**

- a verbal unification claim without a frozen replay
- kernel/action wording that outruns the artifact chain

## Batch 3: 3D Continuum / Asymptotic Closure

**Question**

What actually survives the refinement limit on the strongest ordered 3D
lattice lane?

**Why this matters**

- This is the best route to turning a strong finite-lattice result into a
  defensible continuum statement.
- It is the main way to answer the “small-grid artifact” critique.

**Success criteria**

- a clean `h` ladder with explicit same-family comparisons
- distance-law behavior stated with finite-size corrections, not hype
- a clear boundary between sign persistence, tail shape, and magnitude control

**What counts as progress**

- near-Newtonian finite-lattice behavior becomes better resolved
- the asymptotic statement gets narrower but more defensible
- the note separates what is stable from what is only slice-dependent

**What does not count**

- exact Newtonian wording unless the frozen data really supports it
- broad “all dimensions solved” language

## Recommended Order

1. Generated geometry closure
2. Architecture / action unification
3. 3D continuum / asymptotic closure

That order improves the physics position fastest:

- generated geometry closes the biggest mechanistic gap
- unification reduces the theory split
- asymptotic closure reduces the artifact / finite-size critique

## What to Deprioritize

- more kernel fishing without a new physics question
- more action sweeps without a closure target
- more 4D work before the 3D asymptotics are tighter
- more doc polishing that is not attached to a new retained artifact

## Operating Rule

If a task does not clearly improve one of the three closures above, defer it.
