# Route Patterns

Use route portfolios to avoid tunnel vision and excessive small iteration.
Generate several routes, then choose the route most likely to move the claim
state.

## Route Types

- `constructive theorem`: prove the missing bridge on the retained surface.
- `exact runner`: turn the decisive step into executable or symbolic checks.
- `atlas reuse`: search retained atlas/tool surfaces before inventing new
  machinery.
- `import retirement`: replace a fitted, observed, or admitted value with a
  derivation or bounded support statement.
- `selector/chamber`: resolve or prove obstruction for a branch, chamber,
  sign, or packet-selection ambiguity.
- `literature bridge`: import a known theorem or external context honestly,
  with no hidden derivation claim.
- `no-go/obstruction`: collapse a tempting route family into reusable negative
  evidence.
- `novel construction`: introduce a new mathematical object only when ordinary
  routes cannot close the gate; include a falsifier and artifact plan.

## Dramatic-Step Score

Score each route from 0 to 3 on:

- claim-state upgrade potential;
- import retirement potential;
- ability to close a review blocker;
- artifactability within the runtime;
- novelty or creative leverage when closure needs new structure;
- suitability for a deep stretch attempt from minimal premises;
- risk of overclaim or hidden input, scored negatively.

Prefer routes that can produce one durable output:

- retained or exact support theorem;
- import-retirement audit;
- no-go theorem;
- decisive runner/log;
- bounded demotion that stabilizes the public claim boundary.

## Hard-Problem Pressure

The dramatic-step gate should not reward only easy exits. When a lane has
named residuals, assign a separate **hard-residual pressure** score:

- `0`: restates a known dependency or no-go without trying a new premise.
- `1`: audits a blocker and tightens the import ledger.
- `2`: makes a real first-principles attempt on one named residual and records
  the exact obstruction.
- `3`: introduces a new structure, derivation path, or falsifier that could
  plausibly retire the blocker.

After two cycles scoring `0-1`, the next cycle must target a `2-3` route even
if the expected deliverable is partial progress rather than closure.

Before declaring no route viable, run stuck fan-out across 3-5 orthogonal
attack frames. Examples:

- derive from the smallest algebraic surface, with all phenomenological values
  forbidden;
- search retained atlas/tool surfaces for an already-present semantic bridge;
- attempt a variational, index, Ward, symmetry-breaking, or source-domain
  route;
- prove the strongest impossibility theorem for a full route family;
- build a falsifier that distinguishes two live interpretations.

Record the fan-out in `ROUTE_PORTFOLIO.md` and synthesize why the selected
next route is best.

## Successful Patterns From This Repo

- `Cl(3)/Z^3 -> SM algebra`: decompose the target into exact operator
  subclaims, then land a synthesis note only after matrix-level checks exist.
- EW/top/hierarchy: audit normalizations and keep running/bridge assumptions
  separate from same-surface identities.
- CKM: extract compact theorem surfaces from a broad atlas with explicit
  no-fitted-CKM-input clauses.
- DM/leptogenesis: treat selector/chamber no-gos as map data; do not let a
  target-surface theorem masquerade as global native closure.
- Koide: preserve support and no-go narrowing; reject beautiful closure prose
  when the decisive semantic bridge is missing.
- gravity/wave lanes: use frozen replay and controls before promoting a
  numerical signal.

## Churn Anti-Patterns

Reject or reframe routes that only:

- add smoother prose with the same hidden import;
- create another nearby runner without checking the decisive bridge;
- re-open a no-go route without a new premise;
- review arithmetic while ignoring symbol-to-physics identification;
- weaken wording repeatedly without deciding the claim status;
- chase literature similarities without classifying their import role.

Do not reject a hard stretch attempt merely because it lacks a polished theorem
at checkpoint time. Reject it only if it lacks a named residual, explicit
premise set, falsifiable obstruction, or verification surface.
