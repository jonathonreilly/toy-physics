## Methodology observations from this session (raw, for later cleanup)

**The session itself exhibited several patterns relevant to the methodology paper:**

1. **Open-ended evaluation → focused strategic discussion → execution.** No detailed micro-prompting; the user gave broad direction and the assistant produced structured assessments at each step.

2. **Multi-pass repo audit.** The user requested gap reviews at four separate points (prompts 001, 006, 010, 017, 019), each time triggering a fresh `git fetch` + claims-table re-read + gap recomputation. The cycle compressed the user-side cognitive load while letting the assistant do the data-gathering.

3. **Course-corrections.** Prompts 003 ("sorry why is it incorrect..."), 004 ("fix your hygine note"), and 022 ("we have only been active for 2 months, not 2 years") are explicit course-corrections of prior assistant claims. These are critical methodology evidence: the user catches assistant errors and forces revision rather than accepting smooth-but-wrong output.

4. **Strategy / sociology threading.** Prompts 009, 012, 013, 015, 016 are about reception, co-authorship, and publication strategy — the human-judgment layer that the AI cannot make.

5. **Pivot to methodology framing.** Prompt 023 is the philosophical pivot: "AI will completely change the way science is done — lean in." This shifts the strategic frame from "submit conservatively" to "set the AI-method template." The methodology paper this captures is a direct consequence.

6. **Same-session execution of a large write task.** Prompt 024 launches the methodology MD construction inside the same session that produced the strategic frame for it. The task involves: branch creation, scope analysis, prompt extraction, raw capture across multiple file types, structured output. Demonstrates that an end-to-end "discuss → decide → execute" cycle is one Claude session.
