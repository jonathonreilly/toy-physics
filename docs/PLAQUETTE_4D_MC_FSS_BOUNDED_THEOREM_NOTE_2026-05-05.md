# Plaquette 3+1D Wilson MC Finite-Size-Scaling Bounded Theorem Note

**Date:** 2026-05-05
**Type:** bounded_theorem
**Claim scope:** for the accepted full `3 spatial + 1 derived time` periodic
`SU(3)` Wilson plaquette surface at `beta = 6`, a deterministic seeded
finite-volume Monte Carlo run with controlled autocorrelation, block
jackknife, and two finite-size scaling forms produces a `L -> infinity`
bracket whose diameter and central value are reported by the paired
verifier runner. This is a bounded numerical bracket, not an analytic
closure; the canonical comparator `0.5934` is reported as context only and
is not a fitted target.
**Status authority:** independent audit lane only. This source note is a
bounded numerical bracket and does not supply an audit verdict.
**Primary runners:**
- `scripts/frontier_su3_4d_plaquette_fss_verify_2026_05_05.py` (verifier; the load-bearing assertion runner)
- `scripts/frontier_su3_4d_plaquette_fss_data_2026_05_05.py` (paired data generator)
**Generated artifacts:** `outputs/su3_plaquette_fss_2026_05_05/`

## Question

The salvage from PR #528,
[`PLAQUETTE_4D_MC_SUPPORT_NOTE_2026-05-04.md`](PLAQUETTE_4D_MC_SUPPORT_NOTE_2026-05-04.md),
showed that direct finite-volume Metropolis on the full periodic 3+1D
`SU(3)` Wilson surface at `beta = 6` already lands in the canonical
plaquette region at `L = 3`. That note was an honest *support-only* surface;
it did not run a finite-size analysis and it did not produce a
thermodynamic-limit bracket.

The next bounded question is:

> can a controlled finite-size analysis on the same accepted Wilson surface
> -- with autocorrelation and jackknife errors per volume, and at least two
> independent finite-size scaling forms -- produce a `L -> infinity` bracket
> with a documented error budget, computed (not fitted) from the data?

This note answers that bounded question. It does not answer, and does not
attempt, the analytic identification of `P(beta=6)`. The retained-grade
gates from the existing plaquette stack remain in force.

## Answer

Yes, as a bounded numerical bracket. Three seeded finite-volume datasets at
`L in {3, 4, 5}` are generated by the data runner, persisted as repo-local
JSON artifacts, and consumed by the verifier runner. The verifier computes:

- per-volume integrated autocorrelation time using the Madras--Sokal
  windowed estimator;
- per-volume block-jackknife mean and standard error with block size at
  least `2 * tau_int`;
- two independent finite-size-scaling extrapolations of the per-L means,
  weighted by their statistical errors:
    - `M1 (1/V):    P(L) = P_inf + c1 / L^4`
    - `M2 (1/L^2):  P(L) = P_inf + c2 / L^2`
- a leave-one-out jackknife on the FSS fits to capture the model-by-data
  sensitivity.

The resulting `L -> infinity` bracket and its diameter are computed
quantities. The runner does not assert that the bracket equals any specific
literature value; it asserts only that the protocol succeeds and the
bracket is finite, the per-volume statistics are sufficient, and the two
independent FSS forms agree within their combined errors.

The result is intentionally *bounded*. It strengthens the
`PLAQUETTE_4D_MC_SUPPORT_NOTE` from a single-volume support to a controlled
bracket, without claiming an analytic closure and without entering
retained / promoted / weave language.

## Setup

The data generator uses:

- gauge group `SU(3)`;
- Wilson action at `beta = 6`;
- isotropic four-dimensional `L^4` lattice with periodic boundary conditions
  in all four directions (3 spatial + 1 derived time);
- Metropolis-with-staple link updates: for each link, the staple
    `Sigma_mu(x) = sum_{nu != mu} ( forward staple + backward staple )`
  is computed once per visit, then `n_hits` Metropolis trial updates are
  proposed using the same staple. With the staple convention used in the
  runner, the action contribution of the link is
    `S_link = -(beta/3) Re Tr(U_mu(x) Sigma)`
  and the Metropolis change is
    `delta_S = -(beta/3) Re Tr((U_new - U_old) Sigma)`;
- proposal step size `epsilon` auto-tuned during thermalization to a
  target acceptance of approximately `0.50`, then frozen during measurement;
- a cold start (links initialized to the identity);
- per-volume seeding `np.random.seed(SEED + L)` with module-level
  `SEED = 42`, so each artifact is bitwise reproducible from the
  `(L, SEED)` pair recorded in the JSON.

The verifier uses:

- Madras--Sokal windowed integrated autocorrelation time
  `tau_int = 1/2 + sum_{t=1}^{W} rho(t)`
  with the smallest `W` such that `W >= 5 * max(tau_int, 1/2)`;
- block-jackknife mean and SE with block size
  `block = max(4, ceil(2 * tau_int))`;
- weighted least squares for both FSS fits, with statistical weights
  `1 / sigma_L^2`;
- leave-one-out jackknife of each FSS fit to capture sensitivity to
  individual `L` values.

The canonical comparator value `0.5934` is read from
[`PLAQUETTE_4D_MC_SUPPORT_NOTE_2026-05-04.md`](PLAQUETTE_4D_MC_SUPPORT_NOTE_2026-05-04.md)
and standard Wilson-lattice practice. It is reported only for transparency
and is not used as a fit target. The runner's load-bearing assertions do
not reference it.

The two FSS forms agree at `1.67` sigma in the RSS-combined error budget,
which is well within the standard `3` sigma cross-model tolerance. With
only three lattice volumes, two-parameter FSS fits cannot statistically
discriminate between functional forms; the agreement at `1.67` sigma is
the expected level of cross-model consistency.

The `M1` (`1/V`) bracket has center `0.59464` and `2 SE` half-width
`0.00153`. The canonical context value `0.5934` falls inside this
bracket, at distance `0.00124` from the bracket center. This is reported
for transparency only; it is not used as a fit target and the runner does
not assert it.

Per-volume time-series, autocorrelation windows, FSS fit coefficients,
and leave-one-out arrays are persisted in
`outputs/su3_plaquette_fss_2026_05_05/`.

## What This Supports

A bounded, *protocol-controlled* finite-size bracket for
`<P>(beta=6, SU(3), full 3+1D periodic Wilson surface, L -> infinity)`.
Specifically:

- the per-volume seeded MC data are reproducible from `(L, SEED=42)`;
- the per-volume statistical error is computed under explicit
  autocorrelation control rather than a naive `std/sqrt(N)`;
- two independent FSS forms (`1/V` and `1/L^2`) are computed in parallel
  and required to agree within their combined error budgets;
- the resulting `L -> infinity` bracket is bounded above and below by
  computed numbers, not by a literature target.

This does **not** promote any plaquette-stack note, does **not** establish
an analytic closed form for `P(beta=6)`, and does **not** replace the
independent audit lane.

## Falsifiers

This bounded theorem is falsified if any of the following holds:

- a re-run of the data generator with the documented commands and the
  same `SEED = 42` produces per-volume means outside their reported
  jackknife `2 SE` interval on the same hardware class;
- a longer or higher-`L` MC run shows a `L -> infinity` value outside the
  current `M1` bracket beyond its declared diameter;
- a derivation in another part of the repo demonstrates that the leading
  finite-size correction is not in the `(1/V, 1/L^2)` family at `beta = 6`,
  in which case the agreement assertion `F` is structurally unfounded and
  this note must be retracted to a `support_theorem` claim_type;
- the audit lane finds a reproducibility or load-bearing defect that is
  not statistical fluctuation.
