- or the downstream plaquette PF data.

## Theorem 1: one scalar framework-point value does not determine the class-sector vector

Let `V_n` be the normalized positive class-sector simplex

`V_n = {v in R^n : v_i > 0, sum_i v_i = 1}`,

with `n >= 3`.

Let `L(v)` be any nonconstant scalar observable on `V_n` that fixes one real
number at the framework point.

Then fixing one value `L(v) = c` does **not** determine one unique `v` in
`V_n`.

In particular, there exist distinct strictly positive normalized vectors
`v^(A) != v^(B)` with

`L(v^(A)) = L(v^(B))`,

while some second statistic `M(v)` satisfies

`M(v^(A)) != M(v^(B))`.

So one scalar framework-point observable leaves nontrivial class-sector freedom.

## Corollary 1: same scalar value can still give different compressed boundary class functions

Because

`Z_6^env(W) = <K(W), v_6>`,

two distinct positive normalized vectors with the same scalar framework-point
value can still induce different marked boundary class functions at some
holonomy `W`.

So even a fixed same-surface `P(6)` does **not** by itself determine the full
compressed `W`-dependence.

## Corollary 2: what this means for the live PF seam

The accepted same-surface plaquette value is useful and real, but it is not the
same thing as explicit PF closure.

Even once one scalar framework-point plaquette value is fixed, the live PF seam
still requires explicit evaluation of the class-sector matrix elements that
generate:

- `S_6^env`,
- `eta_6(W)`,
- `rho_(p,q)(6)`,
- and the explicit plaquette framework-point PF data.

## Explicit witness

Take a three-weight truncation with weights

`(0,0), (1,0), (2,0)`.

Define two strictly positive normalized coefficient vectors

`v^(A) = (0.20, 0.60, 0.20)`,

`v^(B) = (0.35, 0.30, 0.35)`.

For the scalar statistic

`L(v) = 0 * v_0 + 1 * v_1 + 2 * v_2`,

both vectors have the same value:

`L(v^(A)) = L(v^(B)) = 1`.

But for the higher statistic

`M(v) = 0 * v_0 + 1 * v_1 + 4 * v_2`,

they differ:

`M(v^(A)) = 1.40`,

`M(v^(B)) = 1.70`.

And because `Z(W)` is recovered from the canonical Peter-Weyl evaluation law,
the two vectors also produce different boundary class functions at generic
marked holonomy `W`.

So one scalar framework-point value does not determine the class-sector PF data.
