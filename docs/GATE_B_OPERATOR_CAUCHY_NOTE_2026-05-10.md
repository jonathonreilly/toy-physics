## One-line read

The operator-Cauchy continuum-bridge method used in the rescaled-NN
harness does **not** extend cleanly to the Gate B family on the tested
axes. The result is a scoped bounded no-go with structural diagnosis,
parallel to the alt-connectivity null companion.

## Rows targeted

This probe addresses two distinct rows that sit on the Gate B /
strong-field harness rather than on the NN lattice:

- `gate_b_connectivity_tolerance_note`
  ([`docs/GATE_B_CONNECTIVITY_TOLERANCE_NOTE.md`](GATE_B_CONNECTIVITY_TOLERANCE_NOTE.md)) --
  a numerical bounded replay on a layered 3D graph
- `restricted_strong_field_closure_note`

  source class

The two rows turn out to have qualitatively different relationships with
the operator-Cauchy method, which is the heart of the diagnosis below.

## Method

Operator-Cauchy continuum-bridge is the following bridge protocol, due to
the rescaled-NN lane:

1. Choose a refinement parameter `x_n` with `x_n -> 0` (or `n -> infinity`)
   and a vector observable `vec(n)` of framework observables on the
   refined harness.
2. Compute the L2 difference of successive Cauchy increments,
   `||vec(n) - vec(n+1)||_2`, and fit it to a power law in `x_n`.
3. If the power exists with the right sign and a good fit, the partial
   sums of Cauchy increments are summable; `vec_n` converges; the
   continuum-limit observable vector exists.
4. Then attempt *identification*: do the components of `vec_inf` match a
   known continuum closed form (per-arm widths, kernel structure, etc.)?

## Three-part experiment

### Part 1 -- lattice-resolution refinement on the ordered family

The `scripts/gate_b_connectivity_tolerance.py` harness has explicit
`N_LAYERS = 13, HALF = 5` module constants that are passed as parameters
into the geometry builders. They are therefore in principle a refineable
lattice-resolution axis.

We refine at `r in {1, 2, 3, 4}` with

```text
N_LAYERS = 12 * r + 1
HALF     = 4  * r
K_eff    = K_BASE / r          (fix physical wavelength)
field_s  = FIELD_BASE / r      (fix physical 1/d Coulomb amplitude)
mass-y   = fixed fractions of HALF (so they stay at fixed physical y)
```

The observable vector is 15-dim: 5 observables
`[y_free, y_mass, shift, gain, fpm]` at 3 physical mass-y fractions
`{0.4, 0.6, 0.8} * HALF`.

#### Result

| pair | `||vec(r1) - vec(r2)||_2` | max-component-diff |
|---|---:|---:|
| `r=1 -> 2` | `1.014649e-02` | `7.121e-03` |
| `r=2 -> 3` | `1.123790e-02` | `1.120e-02` |
| `r=3 -> 4` | `6.401634e-04` | `6.292e-04` |

```text
Fit on 3 pts:  ||vec(r) - vec(r+1)||_2 ~ C * (1/r)^p
  p   = +2.7869  (Cauchy if p > 0.5 AND R^2 >= 0.85)
  C   = 4.2035e-02
  R^2 = 0.5998
  Monotone decreasing: NO
  GATE p > 0.5 AND R^2 >= 0.85 AND monotone: FAIL
```

The nominal exponent `p = 2.79` is large but the fit `R^2 = 0.60` is well
below the 0.85 gate, and the sequence is *not* monotone decreasing -- the
`r=2 -> 3` increment is larger than the `r=1 -> 2` increment, after which
the `r=3 -> 4` increment drops by an order of magnitude. This is the
signature of a harness whose lattice discretization does not converge to a
single continuum operator on this observable basis: the small-`r`
increments capture coarse-grid finite-size effects rather than a
continuum-limit residual.

### Part 2 -- ensemble refinement on the jittered family

For the jittered-but-fixed-connectivity sub-family of the same harness,
we measure 64 seeds at `jitter = 0.30` (the worst-noise point of the
source-note's jitter sweep, where `mean_delta` actually changes sign), and
compute the seed-ensemble-mean 15-dim vector at `N in {2, 4, 8, 16, 32, 64}`.

#### Result

| pair | `||vec(N1) - vec(N2)||_2` |
|---|---:|
| `N=2 -> 4`   | `1.464839e+00` |
| `N=4 -> 8`   | `7.616884e-02` |
| `N=8 -> 16`  | `8.409639e-01` |
| `N=16 -> 32` | `2.358620e-01` |
| `N=32 -> 64` | `1.646972e-01` |

```text
Fit on 5 pts:  ||vec(N) - vec(2N)||_2 ~ C * N^r
  r   = -0.4675  (Cauchy if r < -0.4 AND R^2 >= 0.85)
  C   = 1.0114e+00
  R^2 = 0.1798
  Monotone decreasing: NO
  GATE r < -0.4 AND R^2 >= 0.85 AND monotone: FAIL
```

The nominal slope `r = -0.47` is fortuitously close to the CLT prediction
`-1/2`, but the fit `R^2 = 0.18` is catastrophic and the sequence is
non-monotone -- the `N=4 -> 8` and `N=8 -> 16` increments straddle the
trend by more than an order of magnitude. This means the jitter ensemble
does not behave like an i.i.d. CLT-style sample average in this observable
basis; the underlying mass-window-gain response and `F~M` slope are not
fluctuations about a well-defined ensemble mean.

### Part 3 -- restricted-strong-field closure: method mismatch

The closure note's component runners

- `scripts/frontier_sewing_shell_source.py`
- `scripts/frontier_oh_static_constraint_lift.py`
- `scripts/frontier_oh_schur_boundary_action.py`
- `scripts/frontier_microscopic_dirichlet_bridge_principle.py`

all use a hardcoded `size = 15` cubic box, `R = 4` exterior cutoff, and
an `O_h`-symmetric star-supported source class whose seven parameters
`(x1, x2, mix, lam_e, lam_t, m0, ms)` are tuned to satisfy the closure on
exactly that box. The residuals reported in the source note are already
at machine precision:

```text
sewing identity        : ext_err     = 5.204e-17
static constraints     : max residual = 1.789e-15
Schur stationarity     : flux_err    = 9.021e-17
Dirichlet trace match  : trace_match = 2.637e-16
```

Operator-Cauchy is conceptually mismatched here:

1. The theorem is an algebraic finite-box identity, not a bounded
   numerical result with a continuum limit.
2. The source-class parameters were tuned to `size = 15`. At `size = 17`
   or `size = 21` the seven numbers `(m0, ms, x1, x2, mix, lam_e, lam_t)`
   would need re-tuning before the local `O_h` closure could be expected
   to hold; this re-tuning is not a continuum-limit operation but an
   algebraic re-parametrization of the source class.
3. The static constraints
   `H_0 psi = 2 pi psi^5 rho` and
   `H_0 chi = -2 pi alpha psi^5 (rho + 2S)`
   are pointwise algebraic identities on the chosen box -- they admit no
   sense of `infinite-box limit residual`.

The right method for that row is *algebraic verification at a larger
`O_h`-symmetric box with a re-tuned source class*, not a Cauchy fit on a
refinement axis. This is documented here as a scope/method note rather
than as a numerical fit, and is recorded so future bridges do not waste
budget attempting the wrong method on this row.

Operator-Cauchy continuum-bridge method does **not** extend cleanly to the
Gate B family. The result is a **sharp bounded null with structural
diagnosis**.

### Structural reasons for the bounded scope

- **Part 1**: refining the lattice resolution while holding `K`, field
  amplitude, and mass-y position fixed in physical units does not produce
  a monotone-decreasing Cauchy increment. The Gate B harness's action
  `S = L*(1 - phi)` and its `theta`-weighted forward propagator are not
  derived from a continuum PDE, so lattice-resolution refinement of the
  discretization does not converge to a continuum operator on this
  observable basis. The wave number `K` and the Coulomb amplitude can be
  rescaled to hold physical wavelength and physical 1/r law fixed, but the
  forward-only `theta`-cone with `exp(-BETA * theta^2)` weighting has no
  continuum counterpart that the discrete propagator approximates -- the
  cone-angle weighting is a structural feature of the discrete harness,
  not a discretization of a continuum operator.

- **Part 2**: ensemble averaging over seeds at the worst-noise point of
  the source-note's jitter sweep does not converge with the CLT-style rate
  (`r ~ -1/2`) on this observable basis. The harness's
  bounded-but-mixed connectivity-recompute response (cf. source note
  table 1, where `mean_delta` changes sign across architectures) is
  reproduced as a *structural* feature of the harness, not as a
  noise-averageable fluctuation. The L2 increment sequence has decade-
  scale non-monotonicity, which is incompatible with CLT-style decay.

- **Part 3**: the restricted-strong-field-closure row is a finite-box
  algebraic theorem; the operator-Cauchy method has no continuum axis to
  attack.

identity that needs algebraic, not Cauchy, bridges.

## What this is not

- A claim that the Gate B family has no continuum limit -- only that this
  specific operator-Cauchy bridge does not find one on the observable
  basis we chose.
- A claim that the strong-field closure theorem is broken -- it is not;
  it is at machine precision on its hardcoded box. The claim is only that
  operator-Cauchy is the wrong method for that row.
- A replacement for `GATE_B_CONNECTIVITY_TOLERANCE_NOTE.md` or
  `RESTRICTED_STRONG_FIELD_CLOSURE_NOTE.md` -- the existing source notes'
  bounded scopes are preserved unchanged.
- A negative result on the rescaled-NN lane; those bounded operator-Cauchy
  source notes are unaffected.

## Why it matters

This probe sharpens the frontier map:

- The operator-Cauchy continuum-bridge method has a known successful bounded
  comparison domain in the rescaled NN lane.
- It has two known bounded-null companions: the alt-connectivity note and
  this Gate B note.
- It has one known *method-mismatch row* (restricted strong-field closure
  in this note).

Future bridge-method exploration on Gate B family rows should not assume
the rescaled-NN protocol transfers; instead, the next useful question is
which discrete construction admits a continuum-limit observable, and
whether the harness can be reformulated to expose one.
