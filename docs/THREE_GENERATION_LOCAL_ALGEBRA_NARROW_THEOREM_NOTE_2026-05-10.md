## Statement

Let `d` be an abstract positive integer. Consider the `d`-dimensional discrete
hypercube `{0, 1}^d` (the corner set of the unit cube in `R^d`, equivalently the
Brillouin-zone corner set on `Z^d` in the staggered-Dirac construction).

**Combinatorial counts:**

```text
(N1)  total corner count           |{0, 1}^d|  =  2^d.
(N2)  corner count at Hamming-weight w
                                   |{x in {0,1}^d : sum_i x_i = w}|
                                                 =  binomial(d, w),
       for each integer 0 <= w <= d.
(N3)  partition identity (binomial sum)
                                   sum_{w=0}^{d} binomial(d, w)  =  2^d.
```

**Conclusion at `d = 3`:**

```text
(R1)  total corner count                      =  2^3              =  8,
(R2)  weight-0 count   binomial(3, 0)         =  1,
(R3)  weight-1 count   binomial(3, 1)         =  3,
(R4)  weight-2 count   binomial(3, 2)         =  3,
(R5)  weight-3 count   binomial(3, 3)         =  1,
(R6)  partition         8                      =  1 + 3 + 3 + 1.
```

This special case is shown for sanity, not as a load-bearing input. The algebra
also closes for any other positive integer `d` (e.g. `d = 4` gives
`16 = 1 + 4 + 6 + 4 + 1`, verified in the runner).

## Proof

`(N1)` There are exactly `2` choices for each of the `d` independent integer
coordinates `x_i in {0, 1}`. The product rule gives `2^d` total tuples.

`(N2)` A weight-`w` corner has exactly `w` coordinates equal to `1` and the
remaining `d - w` coordinates equal to `0`. Choosing which `w` of the `d`
coordinates are set to `1` is exactly `binomial(d, w)`.

`(N3)` Sum over `w in {0, ..., d}`: every corner has some weight `w`, so the
sum is the total count by `(N1)`. Alternatively, the binomial theorem gives
`(1 + 1)^d = sum_w binomial(d, w)` directly.

`(R1)`-`(R5)`: substitute `d = 3` into `(N1)`, `(N2)`.

`(R6)`: substitute the four weight counts into `(N3)`. ∎

## Derivable corollaries

```text
(C1)  the weight-1 count and weight-2 count are equal:
          binomial(3, 1)  =  binomial(3, 2)  =  3,
       by symmetry of the binomial coefficient (k <-> d - k).
(C2)  the two extremal singletons add to 2:
          binomial(3, 0) + binomial(3, 3)  =  1 + 1  =  2.
(C3)  the corner-count factorization
          8  =  1 + 1  +  3 + 3
              =  2 * (1)  +  2 * (3)
              =  2 * (1 + 3)
              =  2 * 4.
       (Sanity factorization matching the framing in
       `THREE_GENERATION_STRUCTURE_NOTE` item (1):
       "8 = 1 + 1 + 3 + 3".)
```
