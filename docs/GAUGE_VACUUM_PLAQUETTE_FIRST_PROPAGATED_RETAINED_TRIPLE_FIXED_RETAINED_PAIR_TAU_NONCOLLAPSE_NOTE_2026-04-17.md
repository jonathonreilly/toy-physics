## Question

After the propagated-retained-triple target theorem, the minimality/current-bank
no-go theorem, and the identity-`Tau` insufficiency theorem, is there one
sharper exact theorem directly at the minimal finite target

`(Z_6^env(W_A), Z_6^env(W_B), Z_6^env(W_C))`?

More precisely:

- if future work somehow fixed the first retained coefficient pair
  `(rho_(1,0)(6), rho_(1,1)(6))`,
- and also fixed the tail mass `Tau_(>1)`,

would the propagated retained triple then be determined on the current bank?

## Answer

No.

That stronger closure still fails on the present exact coefficient-side bank.

Already on the two-orbit higher slice generated by `(0,2)` and `(0,3)`:

- the higher-orbit tail mass is the single affine constraint

  `Tau_(>1) = 72 u + 200 v`,

- but the three named sample remainders are

  `R(u,v) = u r_(0,2) + v r_(0,3)`,

  with `r_(0,2)` and `r_(0,3)` not proportional to their identity weights,

- so fixing `Tau_(>1)` still leaves nontrivial variation in the propagated
  three-sample remainder,
- and therefore still leaves nontrivial variation in the propagated retained
  three-sample triple once the retained pair is held fixed.

So the branch can now say the sharper thing:

- even exact closure of `(rho_(1,0)(6), rho_(1,1)(6), Tau_(>1))`
  would still not determine the propagated retained triple on the current
  coefficient-side bank.

This is stronger than the earlier identity-`Tau` insufficiency theorem, because
the retained pair is now fixed rather than allowed to slide along an affine
fiber.

## Theorem 1: fixed retained pair and fixed tail mass still do not collapse the propagated triple on the current coefficient-side bank

Restrict the higher-orbit tail to the conjugation-symmetric two-orbit slice

`{(0,2), (0,3)}`.

Let `u, v >= 0` be the corresponding higher-orbit coefficients.

Then:

1. the tail mass is

   `Tau_(>1) = 72 u + 200 v`,

   because

   `m_(0,2) d_(0,2)^2 = 2 * 6^2 = 72`,

   `m_(0,3) d_(0,3)^2 = 2 * 10^2 = 200`;

2. the three named higher-orbit remainder triple is

   `R(u,v) = u r_(0,2) + v r_(0,3)`,

   where

   `r_(0,2) = (-3.893080185857..., -15.156110292326..., -11.519233136320...)`,

   `r_(0,3) = (-1.522409349774..., -32.119989052505..., -28.565642700537...)`;

3. the normalized row profiles are not equal:

   `r_(0,2) / 72 != r_(0,3) / 200`.

Therefore `R(u,v)` is not constant on affine slices of fixed tail mass

`72 u + 200 v = Tau_0`.

So even after fixing the retained pair
`(rho_(1,0)(6), rho_(1,1)(6))` and fixing `Tau_(>1) = Tau_0`, the propagated
retained three-sample triple still varies on the current coefficient-side bank.

## Corollary 1: explicit fixed-retained-pair/fixed-`Tau` witness pair

Take the fixed retained pair

`rho_(1,0)(6) = 0`,

`rho_(1,1)(6) = 0`.

Choose the two higher-orbit coefficient stacks

`P: (u,v) = (1/40, 0)`,

`Q: (u,v) = (0, 9/1000)`.

Then both have the same tail mass

`Tau_(>1)(P) = 72/40 = 9/5 = 1.8`,

`Tau_(>1)(Q) = 200 * 9 / 1000 = 9/5 = 1.8`.

But their propagated three-sample triples are different:

`Z_hat(P) = (0.902672995354..., 0.621097242692..., 0.712019171592...)`,

`Z_hat(Q) = (0.986298315852..., 0.710920098527..., 0.742909215695...)`.

So already on an explicit two-orbit higher slice, fixed retained pair together
with fixed `Tau_(>1)` still leaves distinct propagated triple values.

## Corollary 2: this is sharper than identity-`Tau` insufficiency

The earlier identity-`Tau` insufficiency theorem allowed the retained pair
itself to vary along an affine line.

The present theorem fixes that pair and still finds noncollapse directly in the

So the current bank no longer merely says:
