## Statement

Let `p` and `c` be abstract positive integers. Consider a representation literal
`(p, c)` labelling a representation of a direct-product gauge group
`G_1 x G_2 = SU(N_1) x SU(N_2)`, with the convention that the first slot is the
SU(N_1)-rep dimension and the second slot is the SU(N_2)-rep dimension.

**Definitions** (operational read-offs):

```text
(D1)  n_pair    :=  dim_G1((p, c))  =  p,
(D2)  n_color   :=  dim_G2((p, c))  =  c,
(D3)  n_quark   :=  n_pair * n_color  =  p * c.
```

**Conclusion.** Under `(D1)`-`(D3)`:

```text
(R1)  n_pair      =  p,
(R2)  n_color     =  c,
(R3)  n_quark     =  p * c,
(R4)  A^2         :=  n_pair / n_color  =  p / c.
```

The framework-specific instance `(p, c) = (2, 3)` (with `n_quark = 6` and the
framework `Q_L : (2, 3)` matter content of
`LEFT_HANDED_CHARGE_MATCHING_NOTE.md`)
gives the rational closed-form readout

```text
n_pair  =  2,
n_color =  3,
n_quark =  6,
A^2     =  2/3.
```

This special case is shown for sanity, not as a load-bearing input. The algebra
also closes for any other positive integer pair (e.g. `(3, 5)` gives
`(n_pair, n_color, n_quark, A^2) = (3, 5, 15, 3/5)`, verified in the runner).

## Proof

`(R1)`, `(R2)`: pure definitional read-offs. The rep-literal `(p, c)` carries
its two slots; reading the first slot returns `p` and reading the second slot
returns `c`. This is a direct consequence of the operational definitions
`(D1)`, `(D2)`.

`(R3)`: pure arithmetic. From `(D3)`, `n_quark = n_pair * n_color`. Substitute
`(R1)` and `(R2)`: `n_quark = p * c`.

`(R4)`: pure rational arithmetic. From the definition `A^2 := n_pair / n_color`
and `(R1)`, `(R2)`: `A^2 = p / c`. ∎

## Derivable corollaries

```text
(C1)  A^4  =  (n_pair / n_color)^2  =  p^2 / c^2.
(C2)  n_quark / n_pair  =  c  =  n_color    (cross-check of (D3) consistency).
(C3)  n_quark / n_color  =  p  =  n_pair    (cross-check of (D3) consistency).
```

`(C1)` is the squared form used in CKM and EW closure chains (e.g. the bridge
identity `sin^2(theta_W)|_lattice = A^4` at the framework instance is
4/9 = p^2/c^2|_{p=2, c=3}; see broad note for the load-bearing route).
