# Moonshot Frontier Portfolio

**Date:** 2026-04-05  
**Status:** active shortlist of high-upside moonshot lanes after the wide-lattice
distance-law replay and retained grown trapping transport replay

## Purpose

This is not a claim note.

It is the current shortlist of moonshot directions that are still worth real
effort because they could change the external-interest tier if they work.

Each lane must keep:

- one minimal observable
- one exact reduction or baseline check
- one fast falsifier

## Current ranking

1. **Retained-grown propagating field**
2. **Retained-grown trapping / frontier architecture**
3. **Persistent object plus adaptive readout**
4. **Radically different generated-family geometry**
5. **Later unification pass only if one of the above survives**

## 1. Retained-grown propagating field

Why this is first:

- the exact-lattice wavefield lane is already the strongest moonshot positive
- the simplest grown-family memory blend is now a clean no-go
- that means a stronger generated-family causal-field result would be a real
  tier jump, not another replay

Minimal observable:

- a detector-line phase ramp, lag, or escape effect on the retained Gate B row
  that is clearly stronger than static deflection alone

Required reduction:

- exact recovery of the retained grown baseline at zero coupling

Fast falsifier:

- if the new rule still behaves like smoothing and produces no phase/escape
  observable, freeze it as another bounded no-go and stop tuning that family

## 2. Retained-grown trapping / frontier architecture

Why this is second:

- the grown row now has a real monotone escape observable with exact reduction
- but it is still only a bounded transport probe
- the next real step is to see whether the same family can support a stronger
  frontier observable without overclaiming a horizon theory

Minimal observable:

- one stronger transport-boundary readout than plain escape, for example a
  no-return asymmetry, delayed escape shoulder, or detector-side trapping
  profile that changes qualitatively across one parameter sweep

Required reduction:

- exact recovery of the retained grown baseline at zero trapping strength

Fast falsifier:

- if every variant collapses back to monotone amplitude loss with no sharper
  observable, freeze the lane as transport-only on this family

## 3. Persistent object plus adaptive readout

Why this is third:

- compact repeated-update source objects are now real
- fixed readout localization is robustly closed
- a more dynamical readout is one of the few cheap adjacent variants left

Minimal observable:

- detector/readout support smaller than the broad baseline while keeping a
  usable weak-field mass law on the compact repeated-update object

Required reduction:

- exact zero-source recovery of the free readout

Fast falsifier:

- if an adaptive readout still loses the mass law, freeze readout localization
  on this family and stop spending time on softer static windows

## 4. Radically different generated-family geometry

Why this is fourth:

- the compact generated-family bridge is now robustly closed
- the remaining reopening is not another small bridge tweak
- it would have to be a genuinely different support-width family

Minimal observable:

- one generated family where support is widened enough that the field rule can
  matter again, ideally with sign plus partial mass-law recovery

Required reduction:

- zero-source or zero-field baseline must still match the intended free case

Fast falsifier:

- if support still localizes too hard under a genuinely different geometry
  family, freeze compact-family transfer as geometry-limited and deprioritize
  that lane

## 5. Later unification pass

Do not force this now.

It only becomes worth it if one of the above lanes lands a real retained
observable first.

The most promising later unification questions would be:

- can the exact-lattice wavefield mechanism and the retained-grown trapping
  transport live in one architecture?
- can the compact repeated-update object become the sourced object in a
  propagating-field lane?
- can a new generated geometry family support both wider transport and a
  stronger causal-field observable?

## Current operating rule

If a moonshot lane does not clear one new retained observable or one stronger
no-go, it should not sprawl.

The point of the portfolio is optionality, not narrative inflation.
