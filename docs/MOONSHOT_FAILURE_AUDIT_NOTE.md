# Moonshot Failure Audit

**Date:** 2026-04-05  
**Status:** explicit re-check of closed moonshot lanes before freezing them

## Purpose

This note is the extra adversarial pass required after a moonshot lane closes
as a bounded negative.

The goal is not to reopen everything.
The goal is to make sure we did not miss one cheap adjacent variant before we
freeze the lane as a real no-go.

Audited lanes:

- generated-family bridge
- grown-family causal-memory / retarded-like field
- detector / readout localization
- distance-law / wide-lattice continuum claim

## 1. Generated-family bridge

Primary closure:

- the compact generated-family bridge stays geometry-limited
- support rescue can move the row, but it does not restore the weak-field
  mass law on the tested compact family

Why the fixed-window/taper family still looks closed after re-checking:

- the discriminator already showed support rescue beats the wavefield bridge
- the follow-up geometry-rule probes did not widen support in the way needed
- the downstream-reach fan broadened static support a little, but did not make
  the wavefield lane matter

Adjacent variant check:

- one new geometry family could still be worth testing later
- but no cheap metric-only reopening is visible on the retained compact family

Verdict:

- robust no-go for the current compact generated-family bridge
- revisit only if a genuinely different geometry family appears

## 2. Grown-family causal-memory / retarded-like field

Primary closure:

- the minimal `gamma`-blended grown-family field is a clean no-go
- exact reduction survives, but the detector phase ramp stays flat and the
  escape ratio remains effectively `1.000`

Why this still looks closed after re-checking:

- the exact `gamma = 0` reduction is fine, so the implementation is not just
  broken at the baseline
- but the causal observable never appears on the retained grown row
- the result behaves like same-site memory smoothing, not a genuine
  propagating field

Adjacent variant check:

- a stronger propagating-field architecture is the only obvious next step
- this lane is not reopened by another small memory blend

Verdict:

- robust no-go for simple layer-memory / retarded-like blending

## 3. Detector / readout localization

Primary closure:

- the hard `3x3` readout is a clean no-go
- the Gaussian taper follow-up also fails

Why this still looks closed after re-checking:

- the broad control keeps the weak-field class
- every localized readout variant tested so far destroys the weak-field mass
  law
- the failure is not just a harsh window artifact

Adjacent variant check:

- a much softer or dynamically adaptive readout could be worth testing in a
  new family
- but the current compact repeated-update object does not hide a cheap
  localized detector

Updated status:

- a later adaptive readout probe now shows that the lane is not closed by
  “all non-broad readouts fail”
- a dynamical diffusion-contour readout can preserve `TOWARD` and near-linear
  `F~M` while shrinking support materially
- but it still does not produce a truly compact detector/readout closure

Verdict:

- robust no-go for fixed-window and tapered readout localization
- bounded partial rescue for adaptive readout
- the lane stays open only through more structural adaptive architectures, not
  through more static window-shape tuning

## 4. Distance-law / wide-lattice continuation

Original closure state:

- the compact grown-geometry tail transfer was retained on `main`
- the broader wide-lattice `h^2+T` continuation was still exploratory on
  `main`

Status after re-checking:

- the retained compact `h = 0.25` family already gives the narrow distance-law
  frontier on `main`
- the wide-lattice branch-side claim has now earned its own retained
  script / log / note chain on `main`
- the safe claim has therefore moved upward from compact frontier only to
  compact-plus-wide retained frontier, but still not to full continuum theory

Adjacent variant check:

- the missed cheap variant was the independent wide-lattice replay
- that replay has now been done and retained
- the next open question is no longer whether the frontier exists on `main`
- it is whether the branch can be promoted beyond finite-lattice frontier
  language into a stronger asymptotic statement

Verdict:

- this item is no longer a closed-negative audit target
- the wide-lattice `h^2+T` continuation has graduated to a retained frontier
  result on `main`
- what remains open is the stronger continuum interpretation, not the
  existence of the wide-lattice replay itself

## Final Read

The closed moonshot lanes were re-checked, and only one cheap missed reopening
was found:

- the wide-lattice `h^2+T` replay

That reopening has now been executed and retained on `main`.

The remaining closed moonshot lanes do not appear to have a cheap missed
reopening on the current retained families, except where a later retained
partial rescue is now explicitly documented.

That means the current moonshot portfolio should move forward, not sideways:

- exact-lattice wavefield / causal-field closure
- retained-grown propagating or trapping-field architecture
- persistent object / inertial-response closure, now with adaptive readout as
  the one surviving detector-side bridge
- a genuinely new geometry family if we want to revisit transfer

The distance-law branch is still the clearest recent promoted frontier on
`main`, and the grown trapping/frontier lane is now a second bounded frontier-
style control. Both should stay narrow claim surfaces for now.
