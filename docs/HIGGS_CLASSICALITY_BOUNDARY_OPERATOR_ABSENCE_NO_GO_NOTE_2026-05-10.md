The no-go is intentionally modest: even if the stipulated bare action has no
fundamental scalar and no explicit `lambda phi^4` term, that does not fix the
`MSbar` scalar quartic unless the lattice-to-`MSbar` matching contribution is
also shown to vanish.

## Statement

Let `lambda_bare` denote the coefficient of a fundamental scalar quartic in a
stipulated lattice bare action. If the action has no fundamental scalar field,
then the operator `lambda_bare phi^4` is absent from that action packet.

The Higgs RGE runner, however, consumes a continuum-scheme boundary

```text
lambda_MSbar(M_Pl) = 0.
```

A general matching relation has the form

```text
lambda_MSbar(M_Pl) = Z_lambda * lambda_bare(a^-1) + delta_lambda,
```

where `delta_lambda` is the finite scheme/convention/matching contribution
between the lattice-bare operator basis and the continuum `MSbar` scalar
quartic.

Therefore the implication

```text
operator lambda_bare phi^4 absent
    ==> lambda_MSbar(M_Pl) = 0
```

is not valid unless a separate theorem proves the relevant matching term
`delta_lambda=0` (and fixes the scheme identification). Operator absence alone
only gives the lattice-bare side of the statement.

## Proof

If the fundamental quartic is absent, write `lambda_bare=0` for the coefficient
of that absent operator in the stipulated bare operator basis. Substituting into
the matching relation gives

```text
lambda_MSbar(M_Pl) = delta_lambda.
```

Thus the desired boundary `lambda_MSbar(M_Pl)=0` follows only if
`delta_lambda=0`. Since `delta_lambda=0` is not supplied by operator absence,
the implication fails.

A one-line countermodel is enough:

```text
lambda_bare = 0,
Z_lambda = 1,
delta_lambda = epsilon != 0
```

satisfies the operator-absence premise while giving
`lambda_MSbar(M_Pl)=epsilon`, not zero. Hence the operator-absence premise is
insufficient.

## Safe Consequence

  quartic matching theorem proving the needed finite matching term vanishes or
  quantifying it inside a bounded claim.
