# Review Hardening Backlog

**Status:** repo-facing backlog note  
**Date:** 2026-04-01

## Purpose

This backlog is not the same as the science frontier. It is the list of work that would make the project more robust to fair external review, especially reviews that focus on circularity, lack of dynamics, weak literature positioning, or unclear derivation status.

## Highest-value backlog

### 1. Assumption-to-derivation ladder

Goal:

- make it impossible to confuse selected ingredients with derived ones

Concrete deliverables:

- [ASSUMPTION_DERIVATION_LEDGER.md](/Users/jonreilly/Projects/Physics/docs/ASSUMPTION_DERIVATION_LEDGER.md)
- one shorter table version in any paper/manuscript

Why it matters:

- this directly answers the strongest “you assumed the physics” critique

### 2. Bounded evolving-network dynamics prototype

Goal:

- address the reviewer criticism that the implementation does not yet fully realize Axiom 1 (“an evolving network”)

Concrete deliverables:

- one minimal graph-growth / event-creation rule
- one bounded demo showing that the geometry later analyzed is actually generated by that rule
- one note on what is and is not yet derived from those dynamics

Why it matters:

- this is the single strongest substantive gap still visible to external reviewers

### 3. Continuum / asymptotic bridge card

Goal:

- separate finite-size artifacts from potentially retained discrete structure

Concrete deliverables:

- one compact scaling note on:
  - isotropy vs connectivity
  - threshold effects vs graph size
  - which current effects appear to survive, weaken, or vanish with size

Why it matters:

- it directly answers “is this just a small-grid artifact?”

### 4. Prediction / discriminator card

Goal:

- make the project testable even at the toy-model level

Concrete deliverables:

- one short note listing:
  - what would falsify the current unitary architecture
  - what would falsify the current decoherence program
  - what toy-level signatures would distinguish this architecture from simpler baselines

Why it matters:

- reviewers trust projects more when they state what would change their mind

### 5. Literature-positioning note

Goal:

- make the repo feel externally aware, not internally self-contained

Concrete deliverables:

- [LITERATURE_POSITIONING_NOTE.md](/Users/jonreilly/Projects/Physics/docs/LITERATURE_POSITIONING_NOTE.md)
- explicit future comparisons to:
  - quantum measure / Sorkin
  - open quantum systems / collision models
  - causal-set dynamics
  - CDT / discrete continuum-limit work

Why it matters:

- it addresses the “insular” criticism without pretending equivalence

## Secondary backlog

### 7. Reproducibility tightening

Goal:

- reduce review friction from underpowered or mismatched artifacts

Concrete habits:

- pair logs with scripts

- avoid underpowered PASS labels

Goal:

- explain in one page what the project currently is, and what it is not

Why it matters:

- it gives future reviewers the honest framing before they infer one
